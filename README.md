# Song Creator Prototype

A desktop-first Song Creator prototype based on the planning docs in `docs/` and `research/`.

## What is included

- landing screen with preset cards and recent projects
- main composer workspace with strategy sidebar, timeline, chord lane, piano-roll-like grid, inspector, and transport bar
- mock regenerate-section modal
- mock export dialog
- polished static interactions with mock songwriting data
- SongScore v1-backed demo data flow for structure, chords, notes, and playback

## Local commands

- `npm install` installs the local Vite toolchain
- `npm run dev` starts the Vite dev server at `http://127.0.0.1:5173`
- `npm run build` creates a production build in `./dist`
- `npm run preview` serves the production build at `http://127.0.0.1:4173`
- `npm run import:vpo` re-imports the checked-in VPO subset from a local VPO 3 download

## Notes

- The frontend now uses a standard Vite workflow instead of the previous custom build/dev/serve scripts
- Audio sample assets stay in-repo and are referenced with `new URL(..., import.meta.url)`, so Vite can fingerprint and serve them correctly in both dev and production builds
- The VPO integration is intentionally tiny: `scripts/import-vpo-samples.mjs` copies only the sample files needed for `cello-SOLO-sustain` and `flute-SOLO-sustain`, then regenerates `src/audio/vpoCatalog.ts`
- The app remains framework-free browser code organized under `src/`; Vite is used for serving, asset handling, and production bundling

## Project structure

- `src/` app source, mock data, and UI modules
- `index.html` Vite entry HTML
- `src/songscore/` local SongScore model constants and UI/playback adapters
- `src/data/demo-song-score-v1.json` generated SongScore example for the featured demo song
- `research/songscore_v1/` vendored external SongScore v1 references copied into this repo
- `vite.config.ts` Vite configuration for dev, preview, and asset handling
- `scripts/` auxiliary local tooling such as sample generation
- `dist/` production output generated by `npm run build`
